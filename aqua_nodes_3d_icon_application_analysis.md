# Aqua Nodes 페이지: 3D 아이콘/오브젝트 적용 방식 분석

> 분석 기준: 사용자가 제공한 HTML 스냅샷 파일  
> 페이지 제목: `Aqua Nodes`  
> 메타 설명: `생각을 어항에 담다`

---

## 1. 결론 요약

이 페이지는 전통적인 의미의 **3D 아이콘 파일**을 직접 삽입한 구조라기보다,  
**p5.js/canvas 기반의 인터랙티브 오브젝트를 3D처럼 보이게 연출한 방식**에 가깝다.

즉, `.glb`, `.gltf`, `.obj`, Three.js 모델, Spline embed 같은 3D 에셋을 직접 불러오는 흔적은 보이지 않는다.  
대신 화면 중앙 전체를 차지하는 `<canvas>` 안에서 움직이는 노드/물고기/입자형 오브젝트를 렌더링하고,  
그 위에 HTML UI를 얹는 구조로 구성되어 있다.

---

## 2. 전체 구조

페이지는 크게 3개의 레이어로 나뉜다.

```txt
[HTML UI Layer]
- 상단 컬러 필터 버튼
- 검색 버튼 / 검색 input
- 하단 노트 카드
- Notion 버튼
- 추가 버튼

[Interaction Overlay Layer]
- 마우스 플래시 효과
- 로딩 pulse animation
- 노트 패널 / 모바일 카드

[Canvas Visual Layer]
- p5Canvas
- 유영하는 오브젝트
- 어항/수중 공간 같은 메인 비주얼
```

가장 핵심적인 시각 요소는 HTML 요소가 아니라 아래 canvas다.

```html
<canvas id="defaultCanvas0" class="p5Canvas"></canvas>
```

이 canvas가 화면 전체를 채우고 있으며, 사용자는 이 영역을 드래그할 수 있다.

---

## 3. 3D 아이콘처럼 보이는 요소의 정체

### 실제 3D 모델 사용 여부

HTML 안에서 다음과 같은 3D 모델 관련 요소는 발견되지 않았다.

```txt
three.js
GLTFLoader
.glb
.gltf
.obj
model-viewer
Spline iframe
WebGL 직접 선언
```

따라서 이 페이지의 오브젝트는  
**3D 모델 파일을 불러온 것이 아니라, canvas에서 생성된 그래픽 오브젝트**로 보는 것이 적절하다.

---

## 4. 적용 방식 분석

### 4-1. p5.js 캔버스 기반 렌더링

페이지 안에는 p5.js가 생성한 것으로 보이는 canvas가 있다.

```html
<canvas id="defaultCanvas0" class="p5Canvas"></canvas>
```

이 구조는 일반적으로 다음 방식으로 작동한다.

```js
function setup() {
  createCanvas(windowWidth, windowHeight);
}

function draw() {
  background(...);
  drawObjects();
}
```

즉, 아이콘/오브젝트를 이미지 태그로 넣는 것이 아니라  
코드로 좌표, 크기, 색상, 움직임을 계산해서 매 프레임 그리는 방식이다.

---

### 4-2. 전체 화면 canvas 위에 UI를 얹는 구조

canvas는 화면 전체를 차지한다.

```html
<div style="width: 100%; height: 100%; cursor: grab;">
  <canvas id="defaultCanvas0" class="p5Canvas"></canvas>
</div>
```

그리고 UI는 `position: fixed` 또는 `position: absolute`로 canvas 위에 겹쳐진다.

예시:

```html
<nav style="position:absolute; top:20px; left:50%; z-index:30;">
```

```html
<button style="position:fixed; bottom:28px; right:28px; z-index:60;">
```

이 구조 덕분에 사용자는  
**3D/인터랙티브 공간 안에 UI가 떠 있는 것처럼** 느끼게 된다.

---

## 5. 3D처럼 보이게 만드는 시각적 장치

이 페이지는 실제 3D 모델을 쓰지 않아도 3D적인 느낌을 만든다.  
주요 장치는 다음과 같다.

---

### 5-1. 어두운 배경

페이지 전체는 검정에 가까운 배경을 사용한다.

```css
background: rgb(0, 0, 0);
```

검은 배경은 오브젝트가 빛나는 듯한 느낌을 만들고,  
수중 공간이나 우주 공간 같은 깊이감을 준다.

---

### 5-2. 반투명 색상

상단 필터 버튼에는 반투명 컬러가 적용되어 있다.

```css
background: #00aaff55;
border: 1px solid #00aaff80;
```

```css
background: #55f21c55;
border: 1px solid #55f21c80;
```

```css
background: #ff4d8f55;
border: 1px solid #ff4d8f80;
```

이런 alpha 값이 있는 색상은  
오브젝트가 유리, 젤리, 빛, 수중 생물처럼 보이게 만든다.

---

### 5-3. blur / glass 효과

UI 버튼에는 `backdrop-filter: blur()`가 적용되어 있다.

```css
backdrop-filter: blur(8px);
```

하단 버튼에도 blur가 사용된다.

```css
backdrop-filter: blur(4px);
```

이 효과는 canvas와 UI 사이의 경계를 흐리게 만들어  
화면 전체가 하나의 공간처럼 느껴지게 한다.

---

### 5-4. radial gradient 마우스 플래시

마우스 주변에 빛이 따라오는 효과가 있다.

```css
background: radial-gradient(
  200px at -999px -999px,
  rgba(255, 255, 255, 0.18) 0%,
  rgba(255, 255, 255, 0.07) 40%,
  rgba(0, 0, 0, 0) 70%
);
```

이 효과는 사용자가 마우스를 움직일 때  
수중에서 빛이 번지는 듯한 느낌을 만든다.

네 명상 페이지에 적용하면  
사용자가 화면 안의 오브젝트를 탐색하는 느낌을 줄 수 있다.

---

### 5-5. pulse animation

로딩 화면에는 동심원 pulse 애니메이션이 있다.

```css
@keyframes aq-pulse {
  0%   { opacity: 0.9; transform: scale(0.96); }
  50%  { opacity: 0.25; transform: scale(1.04); }
  100% { opacity: 0.9; transform: scale(0.96); }
}
```

이 애니메이션은 오브젝트가 살아있는 것처럼  
천천히 숨 쉬는 느낌을 만든다.

네가 원하는 명상 페이지의 3D 오브젝트에도 이 원리를 그대로 적용할 수 있다.

---

## 6. 아이콘/오브젝트를 페이지에 넣은 방식

### 6-1. 아이콘을 이미지로 넣지 않음

이 페이지는 일반적인 방식처럼 아래처럼 하지 않는다.

```html
<img src="/icon.png" />
```

대신 다음과 같은 방식에 가깝다.

```txt
데이터 / 노트 / 색상 테마
↓
canvas 안에서 오브젝트 생성
↓
사용자 조작에 따라 위치, 움직임, 상태 변경
```

즉, 아이콘은 고정된 이미지가 아니라  
**데이터와 상호작용하는 살아있는 오브젝트**처럼 작동한다.

---

### 6-2. 오브젝트와 데이터의 연결

HTML 내부에는 노트 데이터가 포함되어 있다.

```txt
title
date
excerpt
wordCount
color
content
theme
```

각 노트는 다음과 같은 색상 테마를 가진다.

```txt
noted   → #00aaff
shipped → #55f21c
linked  → #ff4d8f
```

이 색상 체계가 canvas 오브젝트와 UI 버튼에 연결되어 있는 것으로 보인다.

즉, 오브젝트는 단순 장식이 아니라  
노트 데이터의 상태를 시각화하는 역할을 한다.

---

## 7. UI와 3D 느낌의 결합 방식

이 페이지에서 중요한 점은  
3D 오브젝트 자체보다 **UI를 얼마나 얇게 유지하는지**다.

### UI 특징

```txt
- 작은 버튼
- 낮은 opacity
- 얇은 border
- 흐릿한 흰색 텍스트
- 넓은 빈 공간
- canvas를 방해하지 않는 위치
```

예시:

```css
color: rgba(255,255,255,0.2);
border: 1px solid rgba(255,255,255,0.12);
background: rgba(255,255,255,0.06);
```

UI가 강하게 보이지 않기 때문에  
사용자의 시선은 canvas 오브젝트에 머문다.

---

## 8. 모바일 대응 방식

모바일에서는 노트 패널이 하단 카드 슬라이더로 바뀐다.

```css
scroll-snap-type: x mandatory;
```

카드 하나의 폭은 다음처럼 설정되어 있다.

```css
flex: 0 0 72vw;
```

즉, 데스크톱에서는 옆에서 패널이 나오고,  
모바일에서는 하단에서 카드가 올라오는 구조다.

네 사이트에 적용한다면:

```txt
데스크톱 명상 페이지:
- 오른쪽에서 음악/공간 선택 패널 등장

모바일 명상 페이지:
- 하단에서 공간 선택 카드 슬라이드 등장
```

로 변환할 수 있다.

## 9. 네 명상 페이지에 적용하는 방식

### 목표

Aqua Nodes처럼 3D 오브젝트를 직접 보여주되,  
복잡한 앱 UI가 아니라 “공간 안에 들어간 느낌”을 만들어야 한다.

---

### 추천 구조

```txt
Meditation Page

[Canvas / WebGL Layer]
- 유기적인 3D 오브젝트
- 물방울 / 구름 / 세포 같은 형태

[UI Layer]
- 상단 세그먼트: 공간 / 음악
- 하단 시작 버튼
- 최소한의 상태 텍스트

[Interaction Layer]
- 마우스 근처 빛 번짐
- hover 시 오브젝트 반응
- 선택 시 부드러운 pulse
```

## 10. 구현 방향 1: p5.js 방식

이 페이지와 가장 유사하게 구현하려면 p5.js를 사용할 수 있다.

### 장점

```txt
- 구현이 비교적 단순함
- 텍스트/입자/물고기/노드 표현에 적합
- 2D인데도 깊이감 있는 연출 가능
- 사용자 입력과 인터랙션 연결이 쉬움
```

### 단점

```txt
- 진짜 3D 모델 느낌은 약함
- 카메라, 조명, 재질 표현에는 한계가 있음
```

### 적합한 경우

```txt
집중하기 페이지
방해요소 오브젝트
텍스트 조각
떨어지는 모션
노래 선택 노드
```

---

## 11. 구현 방향 2: Three.js / React Three Fiber 방식

명상 페이지를 더 확실한 3D로 만들고 싶다면  
Three.js 또는 React Three Fiber를 쓰는 것이 좋다.

### 장점

```txt
- 실제 3D 오브젝트 사용 가능
- 조명, 재질, 카메라, 깊이감 표현 가능
- 유기적인 blob / 물방울 / 구름 구현 가능
```

### 단점

```txt
- 구현 난이도 높음
- 최적화 필요
- 모바일 성능 고려 필요
```

### 적합한 경우

```txt
명상하기 페이지
Aqua 스타일 3D 공간
물방울 / 유체 / 구름 / 세포 오브젝트
```

## 12. 에이전트에게 전달할 구현 설명

아래 문장을 그대로 개발 에이전트에게 전달하면 된다.

```txt
이 페이지의 메인 비주얼은 일반적인 이미지 아이콘이 아니라, 화면 전체를 덮는 canvas 기반 인터랙티브 오브젝트로 구현한다.

HTML UI는 canvas 위에 position fixed/absolute로 얇게 얹는다.

오브젝트는 실제 3D 모델 파일을 직접 삽입하기보다는, p5.js 또는 Three.js로 생성되는 유기적인 노드/입자/물방울 형태로 만든다.

각 오브젝트는 색상 테마, 크기, 위치, 움직임 값을 가지고 있으며, 천천히 유영하거나 호흡하듯 scale 변화가 있어야 한다.

UI는 강하게 보이지 않도록 낮은 opacity, 얇은 border, blur, 작은 크기를 사용한다.

사용자가 조작하는 버튼과 패널은 메인 오브젝트를 방해하지 않고, 공간 위에 떠 있는 얇은 레이어처럼 보여야 한다.

마우스 이동 시 radial gradient 형태의 빛 번짐 효과를 추가해, 사용자가 어두운 수중 공간을 탐색하는 느낌을 만든다.
```

---

## 13. 네 프로젝트에 바로 적용할 디자인 규칙

### 명상하기 페이지

```txt
- 실제 앱 UI보다 공간을 먼저 보여준다.
- 3D 오브젝트는 화면 중앙이 아니라 화면 전체에 퍼져 있어야 한다.
- 오브젝트는 천천히 움직이고, 너무 많이 튀지 않아야 한다.
- 색상은 검정 배경 + 한두 가지 네온 포인트 컬러로 제한한다.
- 버튼은 아주 작고 얇게 만든다.
- 음악 선택 UI는 하단 또는 우측에서 조용히 나타난다.
- 시작 후에는 UI 대부분을 숨긴다.
```

---

### 집중하기 페이지

```txt
- p5.js/canvas를 활용해 텍스트 오브젝트를 물리적으로 다룬다.
- 방해요소 텍스트는 리스트가 아니라 화면 속 물체처럼 배치한다.
- 드래그, 떨어짐, 충돌, 잠금 애니메이션을 활용한다.
- 집중 모드에서는 숫자 타이포그래피만 크게 남긴다.
- UI는 명상 페이지보다 더 실험적이고 키네틱하게 가져간다.
```

## 14. 핵심 참고 포인트

이 페이지에서 가장 가져와야 할 것은 “3D 아이콘” 자체가 아니라  
다음 4가지 구조다.

```txt
1. 전체 화면 canvas
2. canvas 위에 얇게 얹힌 UI
3. 데이터가 오브젝트가 되는 구조
4. 어두운 공간 + 반투명 색 + blur + pulse로 만드는 깊이감
```

## 15. 최종 한 줄 정리

이 페이지의 3D 아이콘 적용 방식은  
**정적인 3D 아이콘을 배치한 것이 아니라, p5.js canvas 안에서 데이터 기반 오브젝트를 생성하고, 이를 어두운 배경·반투명 색·blur·pulse·마우스 빛 효과로 3D적인 공간처럼 보이게 만든 방식**이다.
